** SpringMVC tutorial **

spring-webmvc 모듈 기반의 [Model2 아키텍처와 MVC 패턴 구조] 어플리케이션 구현

  • case01 : 컨트롤러 bean 등록과 핸들러 메소드의 request 매핑 : @RequestMapping 계열(HandlerMapping)
  • case02 : 핸들러 메소드에서 request header[cookie] 처리 : @RequestHeader(HandlerAdapter)
  • case03 : 컨트롤러에서 뷰를 사용하는 방법(ViewResolver 활용) ContentNegotiatingViewResolver(default json view) accept 헤더를 기준으로 content-type 결정 TilesViewResolver(tiles-configurer 로 definition설정 파일 로딩) : logical view name --> definition name InternalResourceViewResolver : prefix + logical view name + suffix --> forwarding
  • case04 : 컨트롤러에서 뷰로 데이터 전달 방법(Scope 활용 케이스, HandlerAdapter) Model : Call by reference 방식으로 모델 전달 ModelAndView : return Value로 모델과 뷰에 대한 정보 전달 ModelAttribute : - 메소드에 선언 : 해당 메소드는 핸들러 메소드보다 먼저 실행되고, 반환값이 모델에 추가된다 - 핸들러 메소드 아규먼트(객체형)에 선언 : 해당 아규먼트는 모델에 추가됨.(생략가능)
  • case05 : 컨트롤러에서 흐름 제어 방식(forward, redirect) Model : forward로 이동하거나, controller에서 view로 모델 전달하는 용도 RedirectAttributes : flashAttribute 전달용도(redirect이동케이스 )
  • case06 : 핸들러 메소드에서 request parameter 처리 : @RequestParam, Command Object
  • case07 : form UI 구성과 데이터 바인딩(Command Object 의 활용)
  • case08 : json 응답을 전송하는 방법 1. Content-Negotiating view-Resolver : accept 헤더로 json 마샬링 여부 결정 model attribute를 대상을 변환 2. @ResponseBody : 핸들러 메소드의 반환 객체를 대상으로 변환 HttpMessageConverter 에 의해 변환
  • case09 : json 요청을 수신하는 방법 : @ResponseBody
  • case10 : 파일 핸들링(업다운로드)
